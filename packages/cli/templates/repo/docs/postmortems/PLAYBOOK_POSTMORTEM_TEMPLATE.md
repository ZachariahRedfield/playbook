# Playbook Postmortem

Use this repo-scoped postmortem template to reconsolidate what happened without collapsing evidence, interpretation, and doctrine into one narrative.

## Facts

Document only observed evidence here.

- What happened?
- Which commands, logs, artifacts, diffs, timestamps, or outputs support each statement?
- What remains unknown?

## Interpretation

Explain the event using the evidence above.

- Which explanatory hypotheses best fit the facts?
- Where did operator judgment or model assumptions shape the interpretation?
- Which parts are still uncertain or contested?

## Model Changes

Record revised understanding.

- What changed in our mental model of the system, workflow, or failure mode?
- Which prior assumptions no longer hold?
- What invariant, heuristic, or boundary should be updated?

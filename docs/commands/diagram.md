# `pnpm playbook diagram`

## What it does
Generates deterministic architecture visualization artifacts from Playbook-owned command logic.

## Common usage
- `pnpm playbook diagram`
- `pnpm playbook diagram --repo . --out docs/ARCHITECTURE_DIAGRAMS.md`
- `pnpm playbook diagram --deps`
- `pnpm playbook diagram --structure`
- `pnpm playbook diagram system`

## Notable flags and targets
- `--repo <path>`: repository root to analyze (default `.`).
- `--out <path>`: output markdown file (default `docs/ARCHITECTURE_DIAGRAMS.md`).
- `--deps`: include dependency diagram only.
- `--structure`: include structure diagram only.
- `system` target: writes canonical `.playbook/system-map.json` for Observer/UI blueprint rendering.

## System-map refresh policy
`pnpm playbook diagram system` remains the canonical direct generation path.

Playbook also refreshes `.playbook/system-map.json` during `pnpm playbook index --json` (and `pnpm playbook index`) so repository-structure refresh flows keep the system blueprint artifact current without UI-side synthesis.

Rule: Canonical architectural visualization artifacts must be generated by Playbook, not synthesized by the UI.
Pattern: Runtime truth changes -> system-map refresh -> observer reads -> UI renders.
Failure Mode: If the blueprint depends on manual regeneration or UI-owned logic, it will drift from real architecture state.

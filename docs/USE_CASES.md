# Playbook Use Cases

## AI Coding Agents

Use Playbook to ensure AI tools like Codex, Cursor, and other coding agents follow repository architecture rules, documentation standards, and engineering governance contracts.

## Large Engineering Teams

Use Playbook to maintain architecture discipline across large repositories by making governance checks deterministic and CI-friendly.

## Open Source Projects

Use Playbook to prevent documentation drift and enforce repository standards as contributors and maintainers evolve the project.

## Solo Developers

Use Playbook to automatically maintain repository documentation and structure while keeping governance lightweight and repeatable.
